概覽 · Overview

從影像到診斷建議，四個階段

整個流程完整發生在本機端：影像處理、訊號還原、節律分類都跑在使用者的 CPU 或 GPU 上， 只有最後的對話互動會呼叫 OpenAI API。

01

影像前處理

灰階化與 CLAHE 對比增強，降低陰影與紙張皺折的雜訊。

02

訊號數位化

以 hengck23 的三階段模型，把影像還原為 12 導程時域訊號 CSV（500 Hz）。

03

節律分類

boki 量化 1D-CNN 在 5 秒滑動視窗上進行推論，輸出五類心律。

04

對話助理

OpenAI GPT 依分類結果與使用者資料，主動提供衛教、查詢與通報。

—

互動式視覺化

Plotly donut chart 呈現各類別視窗分佈，滑鼠懸停顯示細節。

—

狀態持久化

分析結果存入 session state，聊天過程中不會消失。

設計重點 · Key Design

12 導程進、只取 Lead II 給模型

影像數位化會產出完整 12 導程訊號，但本系統在分類階段只使用 Lead II。 Lead II 的電軸方向最貼近心臟去極化的主向，P 波與 QRS 複合波最清楚， 是臨床 rhythm strip 的標準選擇，也最適合給時序節律模型做判讀。

分類只是起點：把結果變成可行動的建議

系統在產生分類後，會把「主要節律 + 分佈」寫入對話上下文，讓 AI 助理能夠根據使用者的身體狀況、所在位置與飲食偏好， 主動呼叫三個外部工具完成後續動作，而不只是單純回覆文字。

FEATURE 01 · 對話助理

帶診斷上下文的衛教問答

GPT 會看到分類結果（主要節律、各類別視窗佔比、裝置資訊），基於此提供貼合病況的衛教內容。

  • 解釋 AF / VFL / SVTA 等心律的臨床意義
  • 提醒何時需要立即就醫、何時可持續觀察
  • 回答藥物、運動、作息相關問題
openai · gpt-3.5-turbo · function-calling
FEATURE 02 · 就醫媒合

附近心臟科診所查詢

依使用者輸入或定位的地點，搜尋鄰近的心臟科診所與醫院，並列出名稱、地址、評分與距離。

  • 以節律嚴重度決定搜尋半徑與排序
  • 輸出可直接點擊的 Google Maps 連結
  • 支援中文地址與多語系名稱
tool · search_nearby_clinics · Google Maps Places API
FEATURE 03 · 飲食建議

心血管友善食譜

針對心律異常常見的飲食建議（低鈉、低飽和脂肪、富含 Omega-3），由 Spoonacular 檢索真實食譜。

  • 自動套用低鈉 / 地中海飲食等關鍵字
  • 列出熱量、營養素與料理時間
  • 提供食譜外部連結，不偽造內容
tool · search_healthy_recipe · Spoonacular API
FEATURE 04 · 家屬通報

LINE 一鍵推播報告

由助理彙整當次分析摘要，透過 LINE Messaging API 推播到預設的家屬或照護者帳號，不需另外截圖。

  • 包含主要節律、時間戳與簡要建議
  • 僅於使用者明確同意時觸發
  • 支援個別使用者的 TARGET_USER_ID 設定
tool · send_line_report · LINE Messaging API
FEATURE 05 · 資料留存

12 導程 CSV 存檔

數位化結果以上傳檔名為基底輸出 CSV，方便帶去門診、匯入其他分析工具、或做後續研究。

  • 500 Hz · 12 導程完整波形
  • 欄位標準：Time, I, II, III, aVR, aVL, aVF, V1–V6
  • 可做為後續多 Lead 模型的輸入
output · <filename>.csv
FEATURE 06 · 互動視覺化

Plotly 分佈圖 + 狀態持久化

類別分佈用 Plotly donut 呈現，支援滑鼠懸停、縮放；分析結果存入 session_state，切換對話也不會消失。

  • hover 顯示每類視窗數與百分比
  • RWD：手機上自動改為單欄排版
  • 重新提問不會洗掉既有分析
plotly · st.session_state
技術棧 · Stack

組成

FRONTEND

Streamlit

單檔 Python web app，自訂 CSS 與 Plotly 互動圖表。

DIGITIZER

hengck23 3-stage（PhysioNet Submit）

Stage 0/1/2 PyTorch checkpoints，方向校正 → 格點重整 → 訊號萃取。

CLASSIFIER

boki — Quantized 1D-CNN

4 層 Conv1d + 1 層 FC，qnnpack 後端量化；DWT(db1) 預處理。

ASSISTANT

OpenAI GPT · Function Calling

診所查詢（Google Maps）、健康食譜（Spoonacular）、家屬通報（LINE）。
